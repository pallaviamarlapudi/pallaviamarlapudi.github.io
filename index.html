Predictive Modeling of Car Prices Using Regression Techniques

Problem Statement

Accurate car price estimation is a critical task for manufacturers and sellers, yet it is often complicated by the interaction of various vehicle attributes. I aimed to develop a predictive model that quantifies the impact of technical and categorical features on car pricing, using regression-based analytical methods.

Dataset Description

The dataset (carprice.csv) contains 199 observations and 26 variables, each representing characteristics of a unique car model. The target variable is car price, while features include engine size, car body type, brand, fuel type, and performance indicators. Some variables were redundant or had missing values, requiring preprocessing and transformation.

Action

The team used R to perform exploratory data analysis, handled missing values, and applied regression models to determine significant predictors. Techniques included correlation analysis, feature engineering, and model comparison to optimize prediction accuracy.

Result

  • Accurate Price Prediction Models Built - Developed multiple linear and lasso regression models to estimate car prices. Achieved strong predictive accuracy with R² values above 0.85.
  • Key Price Drivers Identified - Engine size, horsepower, and car brand emerged as top predictors. Helped isolate the most influential variables for targeted marketing.
  • Data-Driven Insights Enabled - Regression diagnostics and correlation analysis revealed pricing patterns. Enabled better strategic decision-making based on quantified feature impact.
  • Improved Model Interpretability - Feature selection and regularization enhanced model clarity and performance. Reduced multicollinearity and improved generalizability of results.
  • Customer Value Optimization - Analysis showed trade-offs between price, engine size, and fuel efficiency. Supports recommendations for value-based car selection strategies.

Tools

  • R – for data manipulation, modeling, and visualization
  • ggplot2 – for custom visualizations and boxplots
  • glmnet – for implementing lasso regression
  • dplyr – for efficient data wrangling

Algorithms / Concepts Used

  • Multiple Linear Regression – baseline predictive model for car pricing
  • Lasso Regression – regularization technique to prevent overfitting and perform variable selection
  • Feature Engineering – transforming and encoding variables for model input
  • Correlation Analysis – identifying multicollinearity and variable relationships
  • Model Selection using Adjusted R² & RMSE – choosing optimal models
  • Outlier Detection via Boxplots – ensuring data integrity
  • Data Cleaning and Type Conversion – handling “unknown” values and formatting issues

Medical Insurance Premium Prediction using Predictive Modeling

Problem Statement

Accurate prediction of medical insurance premiums is essential to ensure affordability for policyholders and profitability for insurers. This project develops a data-driven model that analyzes health and lifestyle variables to estimate premium costs. The approach enhances pricing transparency, supports effective risk management, and enables fair premium allocation for policyholders.

Dataset Description

The dataset used is the Medical Insurance Prediction Dataset from Kaggle, comprising 25,000 records and 24 variables representing detailed health, lifestyle, and insurance attributes for individuals in India. Key features include age, BMI, smoking status, chronic disease history, and exercise habits. The target variable is patient_insurance_cost. During preprocessing, patient_year_last_admitted was dropped due to excessive missing data, and 990 rows with missing BMI values were removed to ensure model reliability.

Action (SEMMA Methodology)

  • Sample – Used the Medical Insurance Prediction Dataset from Kaggle with 25,000 rows and 24 variables, capturing health, lifestyle, and insurance data relevant to premium forecasting.
  • Explore – Performed data visualization, multivariate analysis, PCA, and outlier detection to understand variable relationships and distribution patterns.
  • Modify – Conducted preprocessing by removing the patient_year_last_admitted column and 990 rows with missing BMI values. Engineered binary variables for step count (>10,000) and BMI (>80) to better capture extremes.
  • Model – Built and compared models including regression, KNN, bootstrap, neural networks, boosted trees, and ensemble methods. Data was partitioned into training (60%), validation (20%), and test (20%) sets.
  • Assess – Evaluated model performance, identifying the Boosted Tree model as the most effective with R² > 0.95. patient_weight was found to be the most influential predictor.

Result

  • Boosted Tree Model Built – Developed a high-performance predictive model using Boosted Trees to estimate insurance premiums.
  • Achieved High Accuracy – The model attained R² values exceeding 0.95 across training, validation, and test sets, confirming prediction reliability.
  • Key Driver Identified – patient_weight emerged as the most influential factor in premium estimation, surpassing other health metrics.
  • Enabled Strategic Pricing – Model outputs support competitive and fair premium setting, balancing profitability and customer value.
  • Improved Risk Profiling – Enhanced ability to identify high-risk individuals, aiding insurers in more effective financial risk management.

Tools

  • JMP Pro – for data preprocessing, modeling, visualization, and model evaluation
  • Excel – for exploratory data review and formatting
  • Kaggle – as the source platform for the insurance dataset

Algorithms / Concepts Used

  • Boosted Trees – primary model used for accurate premium prediction with high R² performance
  • Neural Networks – tested for capturing complex patterns in the dataset
  • K-Nearest Neighbors (KNN) – used as a non-parametric baseline model
  • Bootstrap Aggregation – applied for model stability and comparison
  • Multiple Regression – used for interpretability and baseline accuracy
  • PCA – applied for dimensionality reduction and variance explanation
  • Multivariate Analysis – to explore relationships between variables
  • Outlier Detection – used to retain valid extreme values impacting predictions

Forecasting Weekly Demand Using Time Series

Problem Statement

Forecasting demand based on temporal and environmental variables is essential for operational planning, pricing strategy, and capacity optimization. This project aims to predict weekly rental counts using time series forecasting techniques enriched with weather data and date-related covariates. The analysis focuses on capturing autocorrelations, identifying external influences, and selecting robust models for accurate multi-variable forecasting.
Autocorrelation, PACF, and white noise tests to validate lag structure and stationarity

Dataset Description

The dataset, sourced from Kaggle, contains 8,760 hourly observations spanning one year with 14 variables. Features include temperature, humidity, solar radiation, date, and holiday indicators. To improve interpretability and focus on long-term patterns, the dataset was aggregated into weekly intervals, reducing noise and enabling effective decomposition into trend and seasonal components.
Trend extraction reveals clear seasonality and demand peaks during warmer periods

Action

  • Data Aggregation – Transformed hourly series into weekly data to improve model stability and visual clarity.
  • Time Series Diagnostics – Applied ACF, PACF, and inverse ACF to determine lag dependencies and ARIMA order selection.
  • Pre-whitening – ARIMA filters were fitted to explanatory variables to remove autocorrelation before cross-correlation analysis.
  • Model Development – Implemented four exponential smoothing models and three ARIMAX configurations (ARIMAX 10, 20, 21) for comparative analysis.
  • Evaluation – Model selection based on MAPE, AIC, and SBC. Residuals analyzed for white noise behavior to validate predictive reliability.
Comparison of ARIMAX models across AIC, SBC, and autocorrelation metrics. ARIMAX 10 selected for best tradeoff.

Result

  • Model Accuracy – Simple Exponential Smoothing model had the lowest MAPE (17.9358), showing strength in univariate forecasting.
  • ARIMAX Performance – ARIMAX10 outperformed more complex models (ARIMAX20, ARIMAX21) with AIC of 660.0482 and SBC of 671.7556.
  • Lag Insights – Cross-correlation identified key variables (e.g., solar radiation, temperature) that influenced target at lag 0 and lag 1.
  • Outlier Detection – All models managed 4–5 outliers effectively, with ARIMAX21 slightly underperforming in generalizability.
  • Deployment Suitability – The final model provides a framework adaptable for real-world demand forecasting where environmental inputs are dynamic.

Tools

  • JMP Pro – for time series decomposition, model fitting, diagnostics, and visualization
  • Excel – for data reshaping, aggregation, and preprocessing prior to modeling
  • Kaggle – data source for hourly weather and rental demand records

Algorithms / Concepts Used

  • Time Series Analysis – including trend-seasonal decomposition and lag diagnostics
  • Autocorrelation and Cross-Correlation – ACF/PACF plots and pre-whitening techniques
  • ARIMA & ARIMAX – models used for capturing internal structure and exogenous effects
  • Exponential Smoothing Models – evaluated across trend types for univariate forecasts
  • Model Selection – based on AIC, SBC, MAPE, and residual noise independence
  • Outlier and Lag Analysis – used for robustness testing and feature relevance interpretation

Classifying Internet Usage Risk Using Time Series Data

Problem Statement

Problematic internet use (PUI) in adolescents poses risks to mental health, sleep, and social well-being. This project focused on building a data-driven classification pipeline capable of identifying children at risk of PUI using passive wearable sensor data and detailed demographic/survey responses. The primary goal was to uncover latent behavioral signals that could guide early intervention strategies for digital wellness.

Dataset Description

The dataset includes 986 labeled samples from the Child Mind Institute’s PUI competition. Each entry links to a parquet file with accelerometer and light-based time-series data (X, Y, Z, ENMO, anglez, light) and a corresponding row in a CSV file with 82 demographic and behavioral columns. The target, sii, was transformed from a multi-class to binary classification label. Time-series features were automatically extracted using tsfresh, merged with survey features via ID joins, and further cleaned to eliminate high-missing or highly-correlated attributes.
Correlation heatmap showing relationships between demographic, survey, and sensor-based features.
High-correlation blocks guided multicollinearity reduction during preprocessing.

Action

  • Time Series Feature Engineering – Applied tsfresh on parquet time series to generate over 130 domain-agnostic statistical features, including energy, entropy, RMS, mean changes, and autocorrelation, across axes and light signals.
  • Preprocessing – Performed categorical encoding using one-hot encoding for survey columns, filled missing values using SimpleImputer, and scaled all numeric values using both MinMax and Standard Scalers for testing impact on model behavior.
  • Feature Selection – Used SHAP value ranking, permutation importance, and correlation heatmaps to reduce dimensionality and retain the most predictive and uncorrelated features for final modeling.
  • Model Development – Trained multiple models including RandomForest for benchmarking and XGBClassifier for optimized performance using grid-based hyperparameter tuning and early stopping on validation folds.
  • Evaluation Strategy – Focused on maximizing weighted F1-score due to mild class imbalance; used stratified train-test split and threshold shifting to align prediction boundaries with optimal sensitivity.
  • Interpretability – Integrated SHAP summary plots, ICE curves, and partial dependence plots to analyze the marginal impact of each feature on model decision boundaries and identify monotonic vs. non-monotonic relationships.
SHAP summary plot highlighting the top 20 most influential features driving model predictions.
Feature value gradients reveal how each input increases or decreases PUI risk.

Result

  • Final Model – Reduced-feature XGBoost model achieved a weighted F1 score of 0.80 on held-out test data, using only the top 20 features ranked by SHAP values, with accuracy also reaching 80% after threshold optimization.
  • Model Comparison – Models using combined time-series and survey features consistently outperformed those using demographics alone, confirming the predictive value of wearable sensor inputs.
  • Key Predictors – Top contributors included internet hours per day, raw sleep disorder scores (SDS), physical height, maximum ambient light exposure, and BMI—all variables with behavioral and physiological relevance.
  • Interpretability Findings – SHAP and PDP visualizations revealed nonlinear relationships for predictors like screen time and SDS score, and ICE curves captured subtle interaction effects between multiple variables.
  • Deployment-Readiness – The model architecture supports integration into health monitoring platforms or educational screening systems, given its ability to interpret results and generalize across individuals.
Pairplot illustrating distributional shapes and interaction patterns among key predictors.
Provides visual validation of non-linear separability and clustering in selected variables.

Tools

  • Python (Google Colab) – used for end-to-end workflow from EDA to model interpretability and final submission generation.
  • TSFresh – extracted 100+ time-series features automatically from X/Y/Z/anglez/light signals for machine learning readiness.
  • XGBoost / Scikit-learn – used for model training, performance evaluation, hyperparameter tuning, and threshold testing.
  • SHAP & pycebox – enabled global and local interpretability using SHAP summary plots, ICE plots, and PDPs for top predictors.
  • pandas / seaborn / matplotlib – core libraries for data wrangling, visualization, and metric plotting.
  • pyarrow – used for efficient reading and batch processing of parquet-formatted time-series sensor files.

Algorithms / Concepts Used

  • Gradient Boosting (XGBoost) – final model chosen for robustness, interpretability, and performance on non-linear data.
  • Random Forest – served as a baseline ensemble to benchmark against XGBoost improvements and gain initial feature insights.
  • Automated Time-Series Feature Extraction – leveraged tsfresh to convert raw signals into statistical features without manual curation.
  • Model Interpretability – used SHAP, permutation importance, ICE, and PDPs to explain how each feature affects prediction probabilities.
  • Threshold Optimization – shifted classification threshold from 0.5 to 0.5109 to maximize F1-score for imbalanced output.
  • Multicollinearity Handling – removed redundant predictors using Pearson correlation analysis and visual heatmaps.
  • F1 Optimization & Class Balance – used weighted F1 score and confusion matrix calibration to ensure recall/specificity balance.

Use Cases

  • Clinical Pre-Screening – Identify adolescents at risk for digital dependency using non-invasive wearable and survey-based data.
  • Health-Tech Integration – Embed model within mobile health platforms or behavioral monitoring tools for real-time feedback.
  • Behavioral Research – Provide evidence linking sensor-recorded physical behavior to screen-time-related psychological outcomes.
  • Educational Policy – Support digital wellness programs in schools by identifying high-risk students and tailoring outreach strategies.
  • Parental Reporting – Translate model insights into interpretable feedback that helps parents monitor tech usage impacts.

Visualizing Continental Energy Trends & Funding Flows in Tableau

Problem Statement

This project aimed to build an interactive Tableau dashboard for analyzing continental energy consumption, renewable generation trends, and investment dynamics from 1990 to 2022. The goal was to enable advanced exploratory analysis using calculated fields, parameterized views, and blended datasets to uncover key demand shifts and investment inefficiencies across technologies and regions.

Dataset Description

The dashboard integrates three structured datasets:
  • Global energy consumption by continent (TWh)
  • Renewable energy generation by region and source (solar, wind, hydro, biofuels)
  • Investment in renewable energy technologies (in billions USD)
Datasets were preprocessed in Tableau Prep for format normalization, null handling, pivoting (wide-to-long), and metadata tagging. Relationships were defined in Tableau's logical layer to preserve row-level detail while allowing flexible analysis across grain levels.
Stacked time-series bar chart revealing Asia’s sharp rise in energy consumption post-2005.

Action

  • Advanced Visual Design – Created layered dashboards combining bar charts, area plots, and packed bubbles with synchronized filtering and custom tooltips.
  • Level of Detail (LOD) Expressions – Used FIXED and INCLUDE LODs to calculate normalized investment-to-generation ratios and region-specific CAGR without altering view granularity.
  • Dynamic Parameters – Integrated user-driven parameter controls to switch between energy types, year ranges, and normalized vs. absolute values.
  • Data Blending with Relationships – Used Tableau’s new relationship model (vs. traditional joins) to combine fact tables across different granularities (e.g., investments vs. generation) while preserving context.
  • Table Calculations – Applied WINDOW_SUM, RUNNING_AVG, and RANK to generate trendlines, moving averages, and region rankings over time.
Area chart split by region and technology with dynamic legends and filterable timelines using dashboard actions.

Result

  • Growth Pattern Discovery – Identified Asia Pacific as the fastest-scaling energy producer, with >4% CAGR in solar and wind since 2010, driven by policy and industrial expansion.
  • Investment Imbalance – Bubble charts exposed disproportionate capital flow into solar and wind, revealing temporal lags (3–5 years) between investment peaks and actual energy yield.
  • Regional Saturation – Used dual-axis charts and rank-index tables to show hydropower flattening in Europe, contrasting with exponential growth curves in Africa and Asia.
  • User Exploration – The final dashboard enabled drill-down exploration by region, year, and energy source, powered by interactive filters, highlights, and context-aware tooltips.
Packed bubble chart with measure-based sizing and color-coded tech segments, revealing capital disparity across renewable types.

Tools

  • Tableau Desktop (2023.3+) – Core platform for advanced dashboard design, interactivity, and calculated analytics.
  • Tableau Prep Builder – Used for pivoting, row-level aggregation, NULL cleansing, and dataset shaping before import.
  • Calculated Fields – Implemented advanced expressions using IF logic, ZN, INDEX(), and WINDOW functions for visual metrics.
  • LOD Expressions – Enabled fixed-rank and context-independent metrics across nested hierarchies (region → continent).
  • Dynamic Parameters & Filters – Empowered end-users to toggle KPIs, normalize metrics, and define visual cutoffs.

Concepts Used

  • LOD Calculations – FIXED/INCLUDE/EXCLUDE expressions for normalized KPIs without affecting granularity.
  • Dual-Axis Synchronization – Used for overlaying investment and generation trends with transparent reference bands.
  • Context Filters & Action Filters – Created responsive dashboards that adapt to user selection hierarchies.
  • Data Relationships (Logical Layer) – Replaced traditional joins with Tableau relationships to retain row-level integrity.
  • Calculated Ranks & Indices – Used RANK_DENSE and INDEX to visualize regional leaders dynamically.

Use Cases

  • Executive Dashboards – Enable policy makers to monitor funding efficiency and generation outcomes regionally.
  • Investor Insights – Help private investors identify optimal allocation by evaluating output per billion dollars invested.
  • Education & Communication – Make complex regional energy data interpretable for academic or public audiences.
